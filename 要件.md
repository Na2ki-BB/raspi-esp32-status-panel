# 要件整理（暫定）

## 全体方針

Raspberry Pi と ESP32 を役割分担して使用する。

- Raspberry Pi は複数サービスを動かす処理側として使う
- ESP32 は画面表示を担当する表示端末として使う
- ESP32 側では大きな処理は行わず、基本的には Raspberry Pi から情報を取得して表示する

---

## Raspberry Pi 側

### 役割

- 複数のサービスを動かす
- 各種処理のメインを担当する

### 接続予定の部品

- BME280
- OLED 1枚目

### OLED 1枚目の用途

- 未定
- 具体的な表示内容は後で決める

---

## ESP32 側

### 役割

- 画面表示を担当する
- Raspberry Pi から情報を取得し、OLEDなどに表示する
- 大きな処理は行わない

### 操作系

ESP32 側には以下の部品を接続する予定。

- OLED
- スライドスイッチ
- タクトスイッチ

### 想定している使い方

- タクトスイッチを押すごとに表示画面を切り替える
- スライドスイッチも何らかの用途で有効活用する

---

## 未定事項

- Raspberry Pi 側のOLEDに何を表示するか
- ESP32 側で表示する画面の種類
- スライドスイッチの具体的な用途
- Raspberry Pi と ESP32 間の通信方法
- ESP32 側の画面更新タイミング
- BME280の取得値をどちらでどう扱うか
